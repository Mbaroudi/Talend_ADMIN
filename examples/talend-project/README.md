# Example: Talend project repository (raw Studio sources)

This is the layout the **TOS CI-builder** (`BUILDER=tos`) expects: the Talend
project exactly as the Studio stores it in its workspace, versioned as-is.

```
my-talend-repo/
├── .gitignore              # copy the one in this directory
├── talend.project          # project descriptor (name = technicalLabel)
├── process/                # jobs: <Job>_<version>.item + .properties
├── context/                # context groups
├── metadata/               # connections, schemas
├── code/
│   └── routines/           # user routines
└── .settings/              # Talend project settings (VERSION these)
```

The project may sit at the repository root (recommended) or one directory
down — the builder finds `talend.project` automatically.

## What to version vs. ignore

| Version (your sources) | Ignore (regenerated by the Studio) |
|------------------------|-------------------------------------|
| `talend.project` | `poms/` (Maven structure, rebuilt at logon) |
| `process/`, `context/`, `metadata/`, `code/` | `.Java/`, `.JETEmitters/` (codegen caches) |
| `.settings/` (project settings) | `.metadata/` (Eclipse workspace) |
| | `.project`, `.classpath`, logs, `temp/` |

The [`.gitignore`](.gitignore) in this directory implements exactly that —
copy it to the root of your Talend repository.
